CS50 第一周作業

CS50 每周作業都稱之為 pset (problem set)，在這裡我會記錄自己實作 pset 的想法，盡可能也回顧一些上課教學的內容。 也希望記下一些英文說法(nomenclature/terminology)，以備將來 google 時派得上用場：

Format code

C 以 printf(“%i", 316); 印出內容，%i 代表要列印的資料型態。

Function 黑箱子

CS50 用黑箱子比喻，這暗示了：大多時候不用去了解黑箱子內的運作細節，只需瞭解如何使用、如何讓人使用。 我猜想用函數可方便與他人協作、或是方便未來的自己維護；往後要學的資料型態、include 別人的檔案，應該都一脈相承：站在巨人的肩膀上，借力使力。

Main function / Helper function

如何協助完成做一碗芒果冰(Main Function)？可能是「備水果料」(Helper Function)。 像這樣把不同功能的黑箱子獨立出來，有助可讀性。 只要看 Main function 這個主要的黑箱子就能大概猜出這檔案想達成的目的是「做一碗芒果冰」，不用在看完一大串細節後才瞭解了解細節是如何運作的，因為把細節都打包到 Helper function 了。 Main function 習慣放在 Helper function 前面，方便閱讀。

Function declaration / Function prototype

為了可讀性習慣將Main function 習慣放在 Helper function 前面，但在 Main function 前面要宣告過 Helper function 在後面會出現。

Pset1: Mario

pset1 要求使用者輸入數字 1~8，並相對應印出高為 1~8 的馬力歐金字塔，若數字不符合1~8，會重新提問，請使用者再次輸入整數。在 Terminal 呈現如下：

                        
                            $ ./mario;
                            Height: -1
                            Height: 0
                            Height: 42
                            Height: 50
                            Height: 4
                               #  #
                              ##  ##
                             ###  ###
                            ####  ####
